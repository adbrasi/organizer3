1. Technology Choices & Why
--------------------------------------------------------------------
GUI Framework     : **PySide6** (Qt 6 ‚Äì LGPL)  
  ‚Äì Modern widgets, CSS-like styling, high-DPI-aware, threaded signals/slots.  
  ‚Äì Ships with a full graphics scene framework (needed for the brush editor).  
  ‚Äì LGPL ‚áí no licence worries; indistinguishable API from PyQt6.

Image/FX Library : **Pillow + OpenCV**  
  ‚Äì Keep Pillow for all existing resizing / watermark code.  
  ‚Äì Use OpenCV only inside the new mosaic/blur tools (fast convolution & pixelate).  
  ‚Äì Both work with NumPy arrays, making brush operations trivial.

Metadata Library : **piexif** for JPEG / WEBP, **Pillow‚Äôs PngInfo** for PNG.  
  ‚Äì Wrap both in a tiny helper so the rest of the code never worries about format differences.

Threading Model  : **Qt Worker-Thread pattern**  
  ‚Äì Long tasks live in a QThread; they emit progress(int), log(str) & finished() signals.  
  ‚Äì UI only connects & reacts ‚áí always responsive.

Packaging         : **PyInstaller ‚Äì-onefile**.  
  ‚Äì Re-use the existing executable workflow.

6. Potential Pitfalls & Mitigations
--------------------------------------------------------------------
PNG metadata size‚ÄÉ‚ÄÉ: Comment chunk limited to < 64 kB.  
  Mitigation: warn the user when json.dumps(meta).encode() > 60 kB.

piexif & WEBP‚ÄÉ‚ÄÉ‚ÄÉ  : piexif cannot write EXIF into WebP on older Pillow versions.  
  Mitigation: require Pillow ‚â• 10.0 (supports WebP EXIF).

External script timeouts: some images may lock up pixivMosaic2.py.  
  Mitigation: subprocess.run(cmd, timeout=120) + retry logic.

Thread-safety: Pillow is not 100 % thread-safe for writes on Windows.  
  Mitigation: keep ThreadPool max_workers = min(8, os.cpu_count()) and open images inside the worker function only.

High-DPI scaling issues: Qt 6 auto-scales, but watermark preview may appear blurry.  
  Mitigation: disable Qt::AA_EnableHighDpiScaling and handle scaling manually if needed.
